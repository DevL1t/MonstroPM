# Monstro Profile Manager (MPM)

> Веб-панель управления профилями софта **Monstro** на базе **PostgreSQL**.
> Просмотр, фильтрация, чистка профилей и аналитика — прямо из браузера.

---

> ⚠️ Это **сторонний инструмент**, не официальный продукт Monstro. Запускать ботов/задачи он не умеет — только управление данными профилей.

---

## Возможности

**Дашборд**
- ключевые метрики: всего профилей, создано сегодня, прогретые, группы, среднее число доменов;
- графики: устройства, платформы, браузеры, профили по группам, возраст профилей, активность, динамика создания за N дней;

**Таблица профилей**
- поиск, сортировка, переключение видимых колонок, горизонтальный скролл;
- состояние таблицы сохраняется в браузере (`localStorage`);
- сортировка по умолчанию настраивается в конфиге (по дате создания, новые сверху).

**Фильтры**
- по группе (чипы), платформе, браузеру, типу устройства, возрасту, активности.

**CRUD**
- создание / редактирование / удаление профиля;
- удобные селекты для платформы, браузера и группы (вместо ручного ввода);
- авто-поля (PID, дата создания, признак мобильного) подставляются автоматически.

**Массовые действия**
- чекбоксы в таблице + «выбрать всё»;
- массовое удаление выбранных (с подтверждением);
- массовая смена группы, включая создание новой группы на лету.

**Настройка и эксплуатация**
- **мастер первого запуска** — настройка доступов к БД и пароля прямо в браузере, без правки PHP;
- **страница «Настройки»** в панели — меняйте параметры (сортировка, прогрев, графики, доступы к БД, пароль) без файлов;
- авторизация по паролю с защитой от перебора;
- адаптивная вёрстка (десктоп / планшет / мобайл);
- **schema-driven**: колонки и типы читаются прямо из БД — поменялась структура таблицы, панель подхватит без правок кода.

## Требования

- **PHP 8.0+** с расширением **`pdo_pgsql`**
- **PostgreSQL** (база Monstro)
- веб-сервер (nginx / Apache) или локальная сборка (OSPanel и т.п.)

---

## Установка

### Шаг 1. Скачать в веб-корень

Склонировать репозиторий в папку, которую отдаёт ваш веб-сервер (`www`, `htdocs`, домен в OSPanel и т.п.):

```bash
git clone https://github.com/DevL1t/MonstroPM.git
cd MonstroPM
```

Либо скачать ZIP с GitHub и распаковать туда же.

### Шаг 2. Включить расширение PostgreSQL для PHP

Панель работает через `pdo_pgsql`. Откройте `php.ini` и убедитесь, что строка раскомментирована:

```ini
extension=pdo_pgsql
```

- **OSPanel:** Меню → выбрать модуль PHP 8.0+ → в его `php.ini` включить `pdo_pgsql`, затем **перезапустить** OSPanel.
- **nginx + php-fpm / Apache:** включить расширение и перезапустить PHP-FPM/веб-сервер.

Проверить, что расширение активно:
```bash
php -m | grep pdo_pgsql
```

### Шаг 3. Открыть в браузере и пройти мастер

Перейти по адресу панели, например:

```
http://ваш-домен/MonstroPM/index.php
```

Если `config.php` ещё нет — откроется **мастер первой настройки** (`setup.php`). Заполните:
- **доступы к PostgreSQL-базе Monstro** (host, port, база, пользователь, пароль);
- **пароль на вход** в панель.

Мастер проверит подключение к БД и сам создаст `config.php` (пароль входа сохраняется **хэшем**). Править PHP-файлы руками не нужно.

> ⚠️ Пока конфига нет, мастер открыт всем — **пройдите его сразу после установки**. После создания конфига мастер закрывается.

После настройки откроется форма входа → введите пароль → список профилей.

> Для headless/CI-деплоя можно создать `config.php` и вручную — структура файла описана ниже в разделе [Конфигурация](#конфигурация-configphp).

### Шаг 4. Дальнейшие настройки

Менять настройки можно прямо в панели — пункт меню **«Настройки»** (логин Monstro, строк на странице, сортировка, условия прогрева, дни графика, доступы к БД, смена пароля). Изменения пишутся обратно в `config.php`.

> При первом заходе панель сама прочитает структуру таблицы из БД (schema-driven) и закэширует её на 10 минут.

### Возможные проблемы

| Симптом | Причина / решение |
|---------|-------------------|
| «Не удалось подключиться к базе» | проверьте доступы в `config.php` и что БД доступна с этого сервера (firewall/`pg_hba.conf`) |
| Белый экран / 500 | не включён `pdo_pgsql` — см. Шаг 2 |
| Открывается код PHP вместо страницы | веб-сервер не обрабатывает PHP — проверьте, что папка отдаётся через PHP-обработчик |

## Конфигурация (`config.php`)

Единственный файл с настройками; создаётся мастером первого запуска. **В репозиторий не коммитится** (в `.gitignore`) — храните его только локально/на сервере.

### Блок `db` — подключение к PostgreSQL
| Ключ | Описание |
|------|----------|
| `host` | хост базы Monstro |
| `port` | порт (обычно `5432`) |
| `dbname` | имя базы |
| `user` | пользователь |
| `pass` | пароль |
| `schema` | схема PostgreSQL (по умолчанию `public`; в формах скрыта — меняйте только для нестандартной схемы) |

### Блок `app` — приложение
| Ключ | Описание |
|------|----------|
| `password` | пароль на вход (открытым текстом **или** bcrypt/argon-хэш `$2y$…`) |
| `monstro_login` | логин в Monstro — для ссылок в меню |
| `table` | таблица профилей (по умолчанию `profiles`) |
| `group_column` | колонка-группа (по умолчанию `party`) |
| `sort_column` | сортировка по умолчанию (`data_create`) |
| `sort_dir` | направление (`desc` — новые сверху) |
| `per_page` | строк на странице |
| `truncate_len` | обрезка длинных значений в ячейках |
| `warm_days` / `warm_domains` | условия «прогретого» профиля (возраст ≥ дней и доменов ≥) |
| `chart_created_days` | за сколько дней строить график «Создано профилей» |
| `age_buckets` | корзины графика «Возраст профилей» |

## Безопасность

- Авторизация по сессии; cookie с флагами **HttpOnly**, **Secure** (на HTTPS) и **SameSite=Lax**.
- **Защита от перебора пароля**: 5 неудачных попыток → блок входа на 10 минут (по IP).
- Пароль можно хранить **bcrypt/argon-хэшем** (поддерживается `password_verify`).
- Все запросы к БД — **подготовленные выражения**; имена таблиц/колонок экранируются.
- Ошибки пишутся в системный temp-лог, наружу отдаётся общий текст — **без утечки SQL/структуры**.
- API закрыт авторизацией (`require_auth_api()`).

## Зависимости

Отдельная установка пакетов **не нужна** — нет ни `composer`, ни `npm`, ни шага сборки.

**Серверная часть** — чистый PHP без фреймворка. Единственное требование платформы:
- **PHP 8.0+** с расширением **`pdo_pgsql`**.

**Фронтенд** — библиотеки подключаются с CDN (версии запиннены, обновляются только вручную):

| Библиотека | Версия | Назначение |
|------------|--------|------------|
| jQuery | 3.7.1 | нужна для DataTables |
| DataTables | 2.1.8 | таблица (+ Responsive 3.0.3, Buttons 3.1.2) |
| Chart.js | 4.4.4 | графики дашборда |
| Lucide | 1.17.0 | иконки интерфейса |
| Font Awesome | 6.5.2 | бренд-иконки (платформы/браузеры) |
| Inter | — | шрифт (Google Fonts) |

## Структура проекта

```
index.php            точка входа: дашборд / профили / настройки (?page=…)
login.php            страница входа
setup.php            мастер первого запуска (создаёт config.php)
api.php              JSON API: данные таблицы, фильтры, статистика, CRUD, массовые действия, настройки
lib.php              БД, интроспекция схемы, авторизация, кэш, конфиг, хелперы
assets/app.js        фронтенд (таблица, графики, формы, массовые действия)
assets/app.css       тема оформления
config.php           ваш конфиг с секретами (НЕ в git; создаётся мастером setup.php)
```

## Производительность

- персистентное соединение PDO (экономит время на удалённую БД);
- файловый кэш схемы (10 мин) и счётчика строк (60 сек);
- статистика дашборда — одним комбинированным запросом;
- таблица — server-side, в браузер едет только текущая страница.
